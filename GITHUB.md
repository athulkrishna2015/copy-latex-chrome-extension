# GitHub Management Guide

This document outlines the workflow for managing the **Copy LaTeX** extension on GitHub.

## 🛠️ Development Workflow

1.  **Clone the Repository**:
    ```bash
    git clone https://github.com/athulkrishna2015/copy-latex-chrome-extension.git
    cd copy-latex-chrome-extension
    ```

2.  **Feature Branches**:
    Always create a new branch for changes:
    ```bash
    git checkout -b feature/your-feature-name
    ```

3.  **Commit Style**:
    Follow the existing commit message style (concise and action-oriented):
    *   `Fix floating copy button toggle race`
    *   `Add configurable rich copy and delimiter settings`

## 📦 Building and Packaging

Before releasing or testing in browsers, run the build scripts to generate the `.zip` files:

*   **Chrome**: `./build-chrome.sh` (Generates `copy-latex-chrome.zip`)
*   **Firefox**: `./build-firefox.sh` (Generates `copy-latex-firefox.zip`)

These scripts handle browser-specific manifest tweaks automatically.

## 🚀 Releasing a New Version

To create a new release on GitHub with both Chrome and Firefox builds:

1.  **Update Version**: Update the `"version"` in `src/manifest.json`.
2.  **Commit and Tag**:
    ```bash
    git add src/manifest.json
    git commit -m "Bump version to v1.x.x"
    git tag v1.x.x
    git push origin main --tags
    ```
3.  **Build and Upload to Releases**:
    Use the GitHub CLI (`gh`) to create a release and attach the ZIP files:
    ```bash
    ./build-chrome.sh && ./build-firefox.sh
    gh release create v1.x.x copy-latex-chrome.zip copy-latex-firefox.zip \
        --title "Release v1.x.x" \
        --notes "List your changes here."
    ```

## 🐛 Issues and Pull Requests

*   **Reporting Bugs**: Use the GitHub Issues tab. Provide the browser version and a link to a site where the issue occurs.
*   **Submitting PRs**: Ensure your code is tested in both Chrome and Firefox. Include a brief description of the fix or feature.
